# 3. 단위 테스트 구조

일반적으로 사용되는 단위 테스트 구조

### 3.1.1 AAA 패턴 사용

Given-When-Then과 유사하다

```
// 준비 구절 Arrange - Given
// 실행 구절 Act - When
// 검증 구절 Assert - Then
```

### 3.1.2 여러 개의 준비, 실행, 검증 구절 피하기

> **Assert 문이 단순히 여러개인 것과 다르다.**

때로는 준비, 실행 또는 검증 구절이 여러개인 테스트를 볼 수 있다.

이러한 최적화 기법은 통합 테스트에서 적용하자

단위 테스트에서 진행할 경우, 여러개의 테스트로 나누는 것이 좋다.

---
* 엄격한 테스트 명명 정책을 시행하지 말라
* 매개변수화된 테스트에 필요한 코드의 양을 줄일 수 있다.
* 검증문 라이브러리를 사용해 가독성을 향상 시킬 수 있다.

해당 장 내용은 테스트 작성시 주의 할 점과 안좋은 사례를 들어 어떻게 바꾸는 것이 좋은지 소개하는 장으로 짜고 있는 테스트가 제대로 되나 의심이 들 때 다시 보자.
